# openclaw-plugin-clawhouse

An OpenClaw plugin that registers ClawHouse as a Gateway channel, enabling the ClawHouse iOS app to connect to your agent and receive real-time state updates.

## What It Does

- Registers a `clawhouse` channel in OpenClaw's Gateway
- Tracks agent state (`idle`, `working`, `out`, `returning`, `chatting`, `resting`, `sleeping`) by hooking into agent and tool lifecycle events
- Exposes Gateway methods for iOS device pairing (`clawhouse.pair`) and state queries (`clawhouse.state`)
- Automatically transitions to `resting` after 5 minutes of inactivity, and `sleeping` after 30 minutes

## Installation

```bash
openclaw plugins install openclaw-plugin-clawhouse
```

## Configuration

After installation, configure the plugin in your OpenClaw settings. The only required field is the pairing token, which must match the token generated by the ClawHouse iOS app.

| Field           | Type   | Description                                      |
|-----------------|--------|--------------------------------------------------|
| `pairingToken`  | string | Token generated by the iOS app for device pairing |

Example configuration in `openclaw.config.json`:

```json
{
  "plugins": {
    "clawhouse": {
      "pairingToken": "your-pairing-token-here"
    }
  }
}
```

## Gateway Methods

The plugin registers two Gateway RPC methods accessible by connected iOS clients:

### `clawhouse.pair`

Pairs an iOS device with the agent.

Request params:

```json
{
  "token": "string",
  "deviceId": "string",
  "deviceName": "string",
  "appVersion": "string"
}
```

### `clawhouse.state`

Returns the current agent state.

Response payload:

```json
{
  "state": "idle",
  "timestamp": 1700000000000
}
```

## Agent States

| State        | Meaning                                              |
|--------------|------------------------------------------------------|
| `idle`       | Agent is online but not doing anything               |
| `working`    | Agent is processing a request                        |
| `out`        | Agent is executing an external tool call             |
| `returning`  | External tool call just completed                    |
| `chatting`   | Agent is sending a message to the user               |
| `resting`    | Idle for more than 5 minutes                         |
| `sleeping`   | Idle for more than 30 minutes                        |

## Development

Clone the repository and install dependencies:

```bash
git clone https://github.com/your-org/openclaw-plugin-clawhouse
cd openclaw-plugin-clawhouse
npm install
```

Build the plugin:

```bash
npm run build
```

Watch mode for development:

```bash
npm run dev
```

Run tests:

```bash
npm test
```

### Project Structure

```
src/
  index.ts    # Plugin entry point, registers channel/hooks/methods
  channel.ts  # Channel definition and state machine
  hooks.ts    # Agent and tool lifecycle hook registrations
  types.ts    # Shared TypeScript types
```
